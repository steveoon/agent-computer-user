## 2. 核心组件

### 2.1 输入层

负责接收和预处理输入数据。

| 组件 | 职责 | 数据来源 |
|------|------|----------|
| 企微消息接收 | 接收用户消息 | 企微 Webhook |
| 对话历史 | 提供上下文 | 客户端 messages 数组 |
| 系统配置 | 提供行为指导 | 系统提示词、工具配置 |

### 2.2 记忆层

管理 Agent 的记忆系统，详见 [wecom-memory-architecture.md](./wecom-memory-architecture.md)。

```
┌─────────────────────────────────────────────────────────────┐
│                        记忆层架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌───────────────────────────────────────────────────────┐ │
│  │              SimplifiedMemoryManager                   │ │
│  │                    (本期实现)                          │ │
│  │  ┌─────────────────────────────────────────────────┐  │ │
│  │  │ loadFromMessages()     从消息加载对话历史        │  │ │
│  │  │ getOptimizedHistory()  Token 预算优化            │  │ │
│  │  │ getStats()             调试统计                  │  │ │
│  │  └─────────────────────────────────────────────────┘  │ │
│  └───────────────────────────────────────────────────────┘ │
│                            ↓                               │
│                    内部委托给                               │
│                            ↓                               │
│  ┌───────────────────────────────────────────────────────┐ │
│  │              CellularMemoryManager                     │ │
│  │                   (底层实现)                           │ │
│  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐   │ │
│  │  │ 短期记忆    │  │ 工作记忆    │  │ 长期记忆    │   │ │
│  │  │ (对话历史)  │  │ (临时状态)  │  │ (本期不用)  │   │ │
│  │  └─────────────┘  └─────────────┘  └─────────────┘   │ │
│  └───────────────────────────────────────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**SimplifiedMemoryManager 职责**：

| 能力 | 方法 | 说明 |
|------|------|------|
| 对话历史加载 | `loadFromMessages()` | 从 UIMessage[] 转换并加载 |
| Token 预算优化 | `getOptimizedHistory()` | 长对话自动裁剪 |
| 统计信息 | `getStats()` | 调试用 |

**不再负责**（改由其他组件处理）：

| 能力 | 原实现 | 现归属 |
|------|--------|--------|
| 事实提取 | SmartExtractor | `classify_intent` 工具 |
| 品牌/位置映射 | 字典匹配 | 后端 API |

### 2.3 推理层

LLM 作为 Agent 的"大脑"，负责理解、推理和决策。

```
┌─────────────────────────────────────────────────────────────┐
│                        推理层架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  输入：                                                      │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 系统提示词  │  │ 对话历史    │  │ 可用工具    │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│         │                │                │                 │
│         ▼                ▼                ▼                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                    LLM 推理引擎                      │   │
│  │                                                     │   │
│  │  1. 理解用户意图                                     │   │
│  │  2. 决定是否调用工具                                  │   │
│  │  3. 生成响应内容                                     │   │
│  │                                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│         │                                                   │
│         ▼                                                   │
│  输出：                                                      │
│  ┌─────────────┐  ┌─────────────┐                          │
│  │ 工具调用    │  │ 文本响应    │                          │
│  │ (可选)      │  │ (最终回复)  │                          │
│  └─────────────┘  └─────────────┘                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**推理策略**：

- LLM 自主决定是否调用工具
- 工具调用结果会反馈给 LLM 继续推理
- 支持多轮工具调用（如：先分类意图，再查询岗位）

### 2.4 工具层

Agent 可调用的能力扩展。

```
┌─────────────────────────────────────────────────────────────┐
│                        工具层架构                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                  classify_intent                     │   │
│  │                  意图分类 + 事实提取                  │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ 输入：当前消息 + 对话历史                             │   │
│  │ 输出：lifecycle_stage + extracted_facts              │   │
│  │ 实现：safeGenerateObject (tool-based pattern)        │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              duliday_job_list_for_llm                │   │
│  │                     岗位查询                          │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ 输入：brandKeyword, locationKeyword, jobTypeKeyword  │   │
│  │ 输出：格式化的岗位列表                                │   │
│  │ 特点：透传用户原话，后端 API 负责映射                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              duliday_interview_booking               │   │
│  │                    面试预约                           │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ 输入：jobId, candidateInfo, preferredTime            │   │
│  │ 输出：预约结果                                        │   │
│  │ 状态：⏳ 后期实现                                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**工具设计原则**：

| 原则 | 说明 |
|------|------|
| 透传优先 | 用户原话透传给后端，不做前端映射 |
| 单一职责 | 每个工具只做一件事 |
| 结构化输出 | 使用 Zod Schema 定义输入输出 |
| 安全包装 | 使用 safeGenerateObject 处理错误 |

### 2.5 输出层

生成最终响应并发送。

| 组件 | 职责 |
|------|------|
| 响应生成 | LLM 生成自然语言回复 |
| 消息发送 | 通过企微 API 发送消息 |
| 状态更新 | 更新会话状态（后期：更新用户画像） |

### 4.2 数据结构定义

#### 请求结构

```typescript
interface ChatRequest {
  messages: UIMessage[];           // 完整对话历史
  systemPrompt: string;            // 系统提示词
  allowedTools: string[];          // 允许的工具列表
  userId?: string;                 // 用户 ID（后期用于用户层记忆）
}
```

#### classify_intent 输出

```typescript
interface ClassifyIntentOutput {
  lifecycle_stage: LifecycleStage;
  extracted_facts: CandidateFacts;
  reasoning: string;               // 分类理由（调试用）
}

type LifecycleStage =
  | 'initial_contact'       // 初次接触
  | 'exploring_needs'       // 了解需求
  | 'job_consulting'        // 岗位咨询
  | 'interview_scheduling'  // 面试安排
  | 'follow_up';            // 后续跟进

// 候选人事实（分两部分）
interface CandidateFacts {
  interview_info: InterviewInfo;   // 面试信息（面试安排阶段收集）
  preferences: Preferences;        // 意向信息（岗位咨询阶段收集）
}

// 面试信息
interface InterviewInfo {
  name?: string;                   // 姓名
  phone?: string;                  // 联系方式
  gender?: 'male' | 'female';      // 性别
  age?: number;                    // 年龄
  applied_store?: string;          // 应聘门店
  applied_position?: string;       // 应聘岗位
  interview_time?: string;         // 面试时间
  is_student?: boolean;            // 是否是学生
}

// 意向信息
interface Preferences {
  brands?: string[];               // 意向品牌（可多选，用户原话）
  salary?: string;                 // 意向薪资
  position?: string;               // 意向岗位
  schedule?: string;               // 意向班次
  cities?: string[];               // 意向城市（可多选）
  district?: string;               // 意向区域
  location?: string;               // 意向地点/商圈
}
```

### 5.2 关键模式

#### safeGenerateObject (Tool-based Pattern)

用于结构化输出，解决 `Output.object()` 的模型兼容性问题。

```typescript
const result = await safeGenerateObject({
  model,
  schema: ClassifyIntentOutputSchema,
  schemaName: 'ClassifyIntentOutput',
  prompt: classifyPrompt,
});

if (!result.success) {
  // 错误处理
  return fallbackResponse;
}

return result.data;
```

#### 透传模式

用户原话直接传给后端 API，不做前端映射。

```typescript
// LLM 提取的用户原话
const facts = classifyResult.extracted_facts;

// 直接透传给后端 API
const jobs = await jobListForLLM({
  brandKeyword: facts.mentioned_brands[0],    // "KFC" 原样传
  locationKeyword: facts.mentioned_locations[0], // "浦东" 原样传
});

// 后端 API 负责映射：
// "KFC" → "肯德基"
// "浦东" → "浦东新区"
```

### 7.2 设计原则

| 原则 | 说明 |
|------|------|
| LLM 优先 | 让 LLM 做理解和推理，规则引擎只做兜底 |
| 透传优先 | 减少前端映射，后端统一处理 |
| 无状态设计 | 每次请求从 messages 重建状态 |
| 渐进增强 | 先实现核心能力，后期迭代增强 |
